1.git status检验文件处于什么状态（git status -s/--short精简状态） 前面两个红色??
2.git add 文件名 为开始跟踪某个文件 前面绿色A
3.git commit -m 提交信息 （提交更新，-m后面的是本次提交的描述）
红色M为（已经提交过的）后已经修改但还没有放进暂存区中
暂存上面红色M这种已修改的，可再次调用git add命令，然后变成绿的M，这个方法是将文件放到暂存区中
git add作用：1.跟踪新文件2.把已跟踪，且已修改的文件放到暂存区3.把有冲突的文件标记成已解决
4.再次运用git commit -m …… 可以提交已暂存的文件
5.git checkout -- 文件名 进行撤销修改，修改成git仓库中所保存的版本
6.git add . （暂存的文件过多用此方法全部添加）
7.git reset HEAD 要移除的文件（这个方法用来从暂存区中移除对应的文件）从绿色M变成红色M
8.git commit -a -m ……跳过暂存区直接提交到Git仓库，修改后或者第一次提交用这个可以跳过git add操作
